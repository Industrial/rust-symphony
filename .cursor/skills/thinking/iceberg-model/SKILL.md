# AI Skill: Iceberg Model Systems Analysis

## Purpose
Give an AI agent the ability to uncover **deeper root causes** of observable events by systematically probing lower levels of system behaviour.

## When to Use
- Diagnosing recurring problems
- Moving from reactive fixes to strategic systemic change
- Root cause analysis in complex environments

## Skill Description
This skill structures reasoning into four layers:
1. **Events** — what is directly observed  
2. **Patterns** — trends or repeated occurrences over time  
3. **Structures** — system relationships and causal connections  
4. **Mental models** — underlying beliefs and assumptions shaping structures

## Input
- A problem description or event
- Historical/temporal context (where available)

## Output
- A multi‑layer analysis identifying deeper problem drivers
- Hypotheses about system constraints and leverage points
- Strategy recommendations targeting underlying causes

## Technique
Iteratively query each level:
1. “What happened?” → describe the event
2. “What trends are visible?” → identify patterns
3. “What connections explain these patterns?” → infer structures
4. “What beliefs or assumptions sustain these structures?” → reveal mental models

Record findings across levels to guide strategic intervention.
